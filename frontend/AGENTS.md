# Frontend Rules

이 문서는 `frontend/` 전체에 적용된다.

상세 설명과 예시는 `../docs/rules/README.md`를 따른다.

## API 연동 규칙

1. 서버 응답은 명시적인 타입으로 다루고 ad-hoc 객체 접근에 의존하지 않는다.
2. API 응답 처리에서 `any`를 기본 해법으로 사용하지 않는다.
3. RFC 9457 에러 응답 해석은 공통 API 클라이언트 또는 공통 유틸에 집중시킨다.
4. 개별 UI 컴포넌트가 fetch 에러 형태를 제각각 가정하지 않는다.

## 구조 규칙

1. 기능 코드는 가능한 한 feature 기준으로 유지한다.
2. 프런트 입력 검증은 UX 개선 수단이며, 최종 진실은 백엔드 검증이다.
3. 백엔드 계약이 바뀌면 타입, API 함수, 화면 사용처를 함께 수정한다.
