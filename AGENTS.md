# Asterum System Rules

이 문서는 저장소 전역 규칙의 기준점이다. 하위 디렉터리에 더 구체적인 `AGENTS.md`가 있으면 그 규칙이 우선한다.

상세 설명과 예시는 `docs/rules/README.md`를 따른다.

## 전역 규칙

1. 새로운 기능 구현은 반드시 별도 git worktree에서 시작한다.
2. worktree는 현재 작업 브랜치를 기준으로 생성한다.
3. worktree 경로는 프로젝트 루트의 `.worktrees/`를 표준으로 사용한다.
4. 권장 worktree 이름 형식은 `.worktrees/<base-branch>-<task-slug>`이다.
5. 문서만 수정하는 사소한 변경이 아니라면 메인 작업 트리에서 직접 기능 구현을 하지 않는다.
6. 백엔드 API 계약이 바뀌면 같은 변경 세트 안에서 프런트 타입과 호출부도 함께 반영한다.
7. 규칙 자체를 변경하면 `docs/rules/README.md`도 함께 갱신한다.

## 테스트 및 검증 규칙

1. 변경 후 검증은 가능한 한 백엔드와 프런트 모두 수행한다.
2. 기존 실패가 있으면 새 변경으로 생긴 실패와 분리해서 기록한다.
3. 테스트 성공 결과 분석 문서는 `docs/test_review/` 아래에 남긴다.
